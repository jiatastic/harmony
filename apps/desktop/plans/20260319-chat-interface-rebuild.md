# 聊天界面重构计划

## 目标

把当前被隐藏的 agent 聊天界面改造成 Harmony 的主交互入口，让非技术用户进入应用后可以直接“提问并得到结果”，而不是先理解终端、工作区、tab 和 agent 命令。

## 当前问题

1. 聊天能力实际上已经存在，但在 `TerminalPanel.tsx` 中被强制回退到终端模式，导致用户看不到真正的聊天体验。
2. 顶部入口以 “AI Workspace” 和 agent 启动器的形式出现，更像开发工具，而不是面向普通用户的聊天产品。
3. 当前中心区域视觉焦点偏向终端和标签栏，聊天区域没有成为产品主舞台。
4. agent 输出虽然能从终端流里捕获，但没有一个稳定、友好的对话容器去承载状态、提示和失败恢复。
5. 对非技术用户不够自解释，缺少明确的空状态、建议动作、可理解的状态反馈和“查看原始终端”的降级路径。

## 改造原则

1. 默认保持终端入口，同时把聊天模式明确可见，不再隐藏。
2. 面向普通用户的文案优先于内部工具术语。
3. 默认路径必须清晰，不需要用户理解命令行。
4. 保留高级能力，但不能让高级能力主导整个界面。
5. 优先复用现有 agent / terminal 基础设施，避免无关的大范围重构。

## 实施步骤

### 1. 解除聊天模式隐藏

- agent tab 创建时默认进入 `terminal` 模式，但保留明确的聊天入口。
- 恢复布局时保留 `agentViewMode`，不再因为历史逻辑而把聊天模式重新隐藏。
- agent tab 的聊天记录继续从现有终端输出流中聚合。

### 2. 做出真正可用的聊天主界面

- 增加独立的聊天面板组件。
- 显示欢迎态、建议问题、消息列表、运行状态、发送区。
- 支持回车发送、空态引导、会话未就绪时禁用发送。
- 明确提供“查看终端 / 返回聊天”的切换，而不是默认暴露终端。

### 3. 重写入口文案和首屏结构

- 把 “AI Workspace” 改成更自然的入口，例如“开始一个对话”。
- 每个 agent 入口增加简单用途说明，减少 CLI / agent 术语暴露。
- 让入口更像产品首页操作，而不是开发者工具条。

### 4. 强化非技术用户体验

- 用自然语言解释当前状态，例如“正在思考”“等待 CLI 响应”“可以继续追问”。
- 把错误和退出态变成可理解的提示，而不是仅显示 session 状态。
- 优化按钮层级、输入框层级、留白和视觉聚焦，让聊天区成为主舞台。

### 5. 验证

- 检查 agent tab 新建、恢复、切换、关闭是否正常。
- 检查聊天消息是否能稳定显示并避免重复回显。
- 检查窄屏和普通桌面尺寸下的布局表现。
- 检查查看终端与回到聊天之间的切换是否可靠。

## 本轮先做

1. 新建本计划文档。
2. 取消隐藏聊天模式，但保持终端为默认入口。
3. 在 `TerminalPanel.tsx` 中接入可发送消息的聊天面板。
4. 更新 `main.css`，把聊天区做成更成熟、清晰、适合普通用户的主界面。

## 后续可继续做

1. 给聊天消息加入更好的 Markdown 渲染。
2. 为不同 agent 做更明确的能力定位和推荐场景。
3. 进一步降低侧边栏和右侧工具面板的视觉权重。
